**File:** `server/services/executor-registry.ts`
**Class:** `ExecutorRegistry` (lines 52-84)
**Registration method:** `register(nodeType: string, executor: NodeExecutor): void`

The registry maps node types to executors using an internal `Map<string, NodeExecutor>` called `executors`. The `register` method stores an executor under its `nodeType` key (`this.executors.set(nodeType, executor)`), and `get(nodeType)` retrieves it later.

It also has two sibling registration methods for special cases:
- `registerConditional(nodeType, executor)` — for branching nodes (IF/Switch) that route items to multiple outputs
- `registerPassThrough(nodeType)` — for sub-node types that carry config but don't execute

Note: the class exists, but `server/services/workflow-executor.ts` doesn't actually use it — `WorkflowExecutor` still dispatches via a long `if (currentNode.type === "...")` chain. So the registry appears to be infrastructure not yet wired into the main executor.
